# Инструкции для Codex

## Суть проекта
Проект **MultiExchange** — это система для мониторинга и управления арбитражными сделками на криптобиржах. Основной сценарий запуска — `app.py`, который поднимает веб‑таблицу статуса и запускает несколько воркеров с логикой арбитража.

## Точки входа
- `app.py` — основной запуск (multiprocessing + web‑grid).
- `modules/arbitrage_manager.py` — orchestration воркеров и ключевая бизнес‑логика.
- `modules/WebGrid_Socket_Polling.py` — веб‑таблица/транспорт.

## Важные папки
- `modules/` — основная логика.
- `source/` — статические данные для работы (описано в `source/readme.txt`).
- `logs/`, `deals_log/`, `modules/deals_history/` — рабочие логи/история, обычно не правим руками.
- `.venv/` — окружение, не трогаем.

## Запуск (локально)
- Установка зависимостей: `pip install -r requirements.txt`
- Запуск: `python app.py`

## Язык, кодировка и именование
- В проекте приняты русские пояснения и комментарии. Комментарии и docstring пишем по‑русски.
- Кодировка файлов: **UTF‑8** без `cp1251`. Стандарт фиксируется в `.editorconfig` для корректной работы в PyCharm на Linux и Windows.
- Если нужно добавить явную кодировку в Python‑файл, использовать `# -*- coding: utf-8 -*-` (только при необходимости).
- Имена файлов и путей — только латиница. Имена переменных/констант/классов — на английском.

## Стиль изменений
- Не править логи и `deals_history`, если это не цель задачи.
- Стараться не менять публичные интерфейсы модулей без явной причины.
